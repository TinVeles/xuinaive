# 🛡️ NaiveProxy Auto-Installer

Полностью автоматизированный bash-скрипт для развертывания собственного сервера **NaiveProxy** на базе веб-сервера Caddy. Скрипт самостоятельно компилирует ядро, настраивает маршрутизацию, генерирует надежные учетные данные и маскирует трафик под обычное посещение сайта `kernel.org`.

**Автор:** [Ilya Rublev](https://www.youtube.com/@Ilya_Rublev)  
**Поддерживаемые ОС:** Ubuntu 22.04 LTS / 24.04 LTS, Debian 12
**Работоспособность скрипта регулярно проверяется на Ubuntu 24.04 LTS**

## ⚠️ Важно: Подготовка перед установкой

Для успешной работы скрипта и автоматического получения SSL-сертификата от Let's Encrypt, **строго необходимо** выполнить следующие шаги до запуска установки:

1. **Арендовать VPS** (минимальные требования: 1 CPU, 1 GB RAM).
2. **Зарегистрировать доменное имя** (подойдет любой дешевый домен в зонах .site, .online, .com и т.д.).
3. **Настроить DNS:** В панели управления вашим доменом создайте **A-запись**, которая будет указывать на IP-адрес вашего арендованного VPS.
4. **Подождать обновления DNS:** Обычно это занимает от 5 до 15 минут. Вы можете проверить привязку, пропинговав свой домен (`ping ваш_домен.com`). Либо через dnschecker.
5. **Иметь действующий Email-адрес:** Он потребуется скрипту для регистрации SSL-сертификата.

## 🚀 Установка

**Подключитесь к вашему серверу по SSH под пользователем `root` и выполните всего одну команду:**

```bash
bash <(curl -Ls https://raw.githubusercontent.com/Rublev13/naiveproxy-instant-install-by-Ilya_Rublev/main/install.sh)
```
### В конце установки скрипт выдаст вам сгенерированные логин и пароль а также подключение для v2rayN в формате json, к слову, логин и пароль можно поменять по вашему желанию, зайдя в Caddy.
## 🛠 Как работает скрипт (Разбор команд)

**Скрипт написан с упором на прозрачность и безопасность. Вот что происходит "под капотом" на каждом этапе:**

### 1. Обновление системы и зависимости
```bash
apt update && apt upgrade -y
apt install -y wget curl nano ufw tar libcap2-bin
```
**Скрипт обновляет пакетную базу ОС и устанавливает базовые утилиты, необходимые для скачивания файлов и настройки фаервола.**

### 2. Оптимизация сети (BBR)
```
net.core.default_qdisc=fq
net.ipv4.tcp_congestion_control=bbr
sysctl -p
```
**Включает алгоритм контроля перегрузок TCP BBR от Google. Это критически важно для прокси-серверов: значительно снижает задержки (пинг) и увеличивает скорость передачи данных, особенно в сетях с потерями пакетов.**

### 3. Установка среды Go (Golang)
NaiveProxy интегрируется в веб-сервер Caddy, написанный на языке Go. Скрипт скачивает официальный архив go1.22.1, распаковывает его в /usr/local/go и добавляет в системные пути (PATH) для работы компилятора.**

### 4. Компиляция Caddy + NaiveProxy
```
xcaddy build --with [github.com/caddyserver/forwardproxy@caddy2=github.com/klzgrad/forwardproxy@naive](https://github.com/caddyserver/forwardproxy@caddy2=github.com/klzgrad/forwardproxy@naive)
setcap cap_net_bind_service=+ep /usr/bin/caddy
```
## Самый ресурсоемкий процесс. 
**Скрипт использует xcaddy для сборки кастомной версии Caddy с внедренным форком модуля forwardproxy от создателя NaiveProxy. Команда setcap выдает собранному файлу права на прослушивание привилегированных портов (80 и 443) без запуска от имени root.**

**5. Изоляция и настройка Caddyfile
Создается отдельный системный пользователь caddy без прав входа в систему (для безопасности). Затем скрипт генерирует файл настроек /etc/caddy/Caddyfile, в котором:**

**Запрашивается SSL сертификат (tls).**

**Включается модуль проксирования (forward_proxy) со скрытием IP (hide_ip, hide_via).**

**Устанавливается перенаправление всех "обычных" запросов (без правильного пароля) на безобидный сайт reverse_proxy https://kernel.org, чтобы скрыть наличие прокси от систем DPI.**

**Генерируются случайные логин (14 символов) и пароль (18 символов).**

**6. Создание системной службы
Скрипт создает файл caddy.service в директории systemd. Это позволяет серверу работать в фоне, автоматически перезапускаться при сбоях и самостоятельно стартовать после перезагрузки VPS.**

**7. Настройка Firewall (UFW)**
```
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 443/udp
ufw --force enable
```
**Включается межсетевой экран. Он жестко блокирует все порты, оставляя открытыми только SSH (для управления сервером) и порты 80/443 (HTTP/HTTPS) для работы веб-сервера и проксирования. UDP 443 открывается для поддержки протокола HTTP/3 (QUIC).**

## 💻 Пошаговая настройка Клиента (Client-side)

В отличие от классических коммерческих VPN, где пользователю достаточно нажать одну большую кнопку, архитектура продвинутых протоколов инкапсуляции базируется на разделении ядра и оболочки. Ядро NaïveProxy запускается в фоне и открывает на компьютере пользователя локальный порт (обычно SOCKS5 порт 1080). Далее пользователю необходимо направить трафик операционной системы или отдельного браузера в этот локальный порт, чтобы ядро зашифровало его и отправило на сервер.

### 🪟 Настройка клиента для Windows (v2rayN)

Наиболее элегантным и удобным решением для операционной системы Windows является использование графической программы-оболочки **v2rayN**. В последних версиях программа получила нативную поддержку интеграции ядра NaïveProxy, позволяя автоматизировать маршрутизацию трафика.

#### Этап 1: Интеграция ядра в оболочку v2rayN
1. Перейдите в репозиторий графического клиента на GitHub по адресу [github.com/2dust/v2rayN/releases](https://github.com/2dust/v2rayN/releases) и загрузите архив последней версии (например, `v2rayN-windows-64-desktop.zip`). Извлеките содержимое архива в удобную директорию (например, `C:\v2rayN`).
2. Перейдите на официальную страницу релизов ядра NaïveProxy: [github.com/klzgrad/naiveproxy/releases](https://github.com/klzgrad/naiveproxy/releases). Загрузите архив с пометкой `win-x64` (к примеру, `naiveproxy-v120.0.6099.43-1-win-x64.zip`). Распакуйте скачанный архив.
3. Скопируйте исполняемый файл `naive.exe` из распакованного архива ядра и поместите его непосредственно в корневую папку оболочки v2rayN (туда, где расположен файл `v2rayN.exe`).

#### Этап 2: Формирование профиля конфигурации
1. Находясь в папке v2rayN, создайте обычный текстовый документ и переименуйте его в `naive.json`. Важно убедиться, что система Windows отображает расширения файлов, и итоговый файл не имеет скрытого расширения `.txt`.
2. Откройте файл в любом текстовом редакторе (например, «Блокнот») и внесите в него структуру подключения:

```json
{
  "listen": "socks://127.0.0.1:1080",
  "proxy": "https://ВАШ_ЛОГИН:ВАШ_ПАРОЛЬ@ВАШ_ДОМЕН.com"
}
```
(Параметры ВАШ_ЛОГИН и ВАШ_ПАРОЛЬ должны строго соответствовать данным, которые выдал скрипт установки. Домен указывается без префиксов типа http:// на конце).

#### Этап 3: Настройка в графическом интерфейсе
Запустите файл v2rayN.exe.

В верхнем меню навигации выберите Servers (Серверы) → Add custom configuration server (Добавить пользовательский сервер конфигурации).

В открывшемся диалоговом окне заполните параметры:

Alias (Псевдоним): Введите любое понятное имя, например, NaïveProxy Server.

Address (Адрес): Нажмите кнопку Browse (Обзор) и укажите путь к созданному файлу naive.json в папке программы.

Core Type (Тип ядра): В выпадающем списке обязательно выберите naiveproxy.

Socks Port: Укажите 1080.

Нажмите ОК. Профиль появится в главном окне списка серверов. Для активации необходимо кликнуть по нему левой кнопкой мыши и нажать клавишу Enter на клавиатуре (профиль выделится цветом или галочкой).

#### Методы маршрутизации трафика (Windows)

Запущенное ядро готово к работе, однако теперь нужно направить в него трафик. Существует три подхода:

Метод 1: Системный прокси (System Proxy). Подходит для большинства пользователей. В системном трее Windows (возле часов) кликните правой кнопкой мыши по синей иконке v2rayN. Выберите меню System proxy и нажмите Set system proxy. Иконка станет красного цвета. В этом режиме запросы большинства популярных браузеров

(Chrome, Edge) будут автоматически перенаправляться в туннель.

Метод 2: Изоляция через расширение браузера (Proxy SwitchyOmega). 

Идеально для случаев, когда туннель нужен только для конкретных сайтов. 

Системный прокси в v2rayN нужно отключить (Clear system proxy). 

Установите в браузер расширение Proxy SwitchyOmega. В его настройках создайте новый профиль (New Profile). Выберите протокол SOCKS5, адрес сервера укажите как 127.0.0.1, а порт 1080. Сохраните настройки (Apply changes) и активируйте профиль через панель браузера.

Метод 3: Захват трафика на уровне драйвера (Tun2Socks). Некоторые программы (игры, Telegram) игнорируют настройки системного прокси. В современных версиях v2rayN интегрирована утилита tun2socks. Достаточно нажать переключатель Tun в нижней части интерфейса программы. Приложение запросит права администратора и самостоятельно поднимет виртуальный сетевой адаптер, принудительно заворачивая абсолютно все пакеты ОС в туннель.

#### 🤖 Настройка клиента для Android (NekoBox)
Экосистема Android предлагает богатый выбор инструментов.

Наиболее универсальным клиентом является приложение NekoBox for Android. Оно обладает модульной архитектурой и поддерживает загрузку дополнительных ядер (плагинов).

Инсталляция клиента: Загрузку рекомендуется производить из официального репозитория разработчика на GitHub: github.com/MatsuriDayo/NekoBoxForAndroid/releases. Загрузите базовый установочный файл (например, NekoBox-1.4.2-arm64-v8a.apk) и установите его.

Инсталляция плагина ядра: NekoBox не содержит протокола NaïveProxy из коробки. В том же репозитории GitHub (в разделе релизов плагинов) найдите APK-файл плагина NaïveProxy и установите его как отдельное приложение. Визуально плагин не имеет интерфейса, но NekoBox распознает его присутствие в системе.

Конфигурация:

Запустите приложение NekoBox. В верхнем правом углу нажмите кнопку создания профиля (иконка плюса/листа).

Выберите Manual input (Ручной ввод конфигурации).

Из списка протоколов выберите NaïveProxy. (Если пункт отсутствует, плагин установлен некорректно).

Заполните поля подключения:

Profile Name: Произвольное название.

Server: Имя зарегистрированного домена.

Port: 443.

Username: Ваш логин.

Password: Ваш пароль.

Сохраните конфигурацию, нажав на галочку.

В главном окне выберите созданный профиль и нажмите плавающую кнопку старта. Согласитесь на создание VPN-подключения в системе Android.

#### 🍏 Настройка клиента для iOS (Shadowrocket)

Операционная система Apple отличается высокой закрытостью. Индустриальным стандартом для iOS-пользователей является приложение Shadowrocket. Несмотря на то, что программа является платной в App Store, она представляет собой мощный маршрутизатор.

Приобретение: Установите Shadowrocket из App Store. (В случае недоступности приложения может потребоваться временная смена региона Apple ID).

Добавление узла конфигурации:

Откройте Shadowrocket. На главном экране нажмите + (Add Server / Добавить сервер).

В меню Type (Тип) выберите протокол HTTP или HTTPS (новые сборки могут поддерживать нативный тип Naive).

Введите технические данные сервера:

Host / IP: Указать доменное имя.

Port: 443.

User: Логин.

Password: Пароль.

TLS / SSL: Если был выбран тип HTTP, обязательно активируйте тумблер шифрования (включить TLS). Без него пароль передастся в открытом виде и будет сброшен сервером.

Нажмите Save (Сохранить) в верхнем правом углу.

Маршрутизация и запуск:

Вернитесь на главный экран. Найдите раздел Global Routing (Глобальная маршрутизация). Для направления всего трафика в туннель выберите режим Proxy. Для использования правил маршрутизации — режим Config.

Включите главный тумблер подключения в верхней части экрана. Подтвердите добавление конфигурации паролем или биометрией.
